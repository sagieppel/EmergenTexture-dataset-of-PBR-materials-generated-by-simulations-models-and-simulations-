
# EmergenTextures Dataset

## General

The **Emergent Textures Dataset** is a large-scale dataset of physics-based rendering (PBR) materials textures generated by scientific models,  simulations and generative code.

The dataset contains **40,000 PBR textures**, generated by **2,000 procedural models and simulations** from diverse areas of science, art, generative coding and procedural generation. These include models from physics, biology, chemistry, engineering, mathematics, art, generative coding, and related fields.

For each model, the Python generation code is provided. Each model can generate an arbitrary number of PBR materials at any resolution, assuming sufficient computational resources.

The dataset was created by collecting and adapting models and simulations from a wide range of scientific and artistic domains. These models generate PBR materials by mapping their internal states, layers, or outputs to standard PBR texture channels, such as:
Base color, Height, Roughness, Metallic, Normals, Transperancy..

Because many of the source models were not originally designed for PBR material generation, the mapping from model outputs to PBR channels is often creative or interpretive.

## Dataset Structure

For each model, we provide **20 generated PBR materials**:

* **10 materials at 512 × 512 resolution**
  See: `PBR512x512...zip`

* **10 materials at 1024 × 1024 resolution**
  See: `PBR1024x1024...zip`

In addition, the code for each PBR-generating model is provided in:

```python
generate.py
```

The generator can be run using:

```python
generate_texture(outdir=..., sz=1024, numsamples=10)
```

This function generates `numsamples` PBR materials at resolution `sz`, by randomly sampling the model’s simulation or parameter space.

Runtime varies significantly between models, ranging from a few seconds to several hours, depending on the simulation and output resolution. Expected runtime information is provided in:

```text
generation_time.json
```

Rendered previews of the generated PBR materials are provided as:

```text
renders...jpg
```

If a material is seamless and tileable, a file named:

```text
SEAMLESS
```

appears in the corresponding folder. Approximately **95%** of the materials in the dataset are seamless.

The PBR materials are organized according to the type of model that generated them, such as: Biology, Physics, Chemistry, Art,Engineering,Creative,General

## Dataset Generation Pipeline

The dataset was generated using an agentic pipeline. Leading large language models, including Kimi, GPT, GLM, Gemini, and others, were instructed to propose methods, simulations, and generative processes that could be implemented or adapted to generate PBR material textures.

The proposed methods were based on either:

* Established scientific, mathematical, and artistic models, such as Navier–Stokes simulations, reaction-diffusion systems, and related processes
* Original generative ideas suggested by the LLM

In all cases, each model had to generate PBR textures by mapping its outputs into PBR texture channels. This mapping is inherently somewhat arbitrary, since most of the underlying scientific and artistic models were not designed natively for PBR material generation.

The code for the agentic dataset-generation pipeline is provided in:

```text
Dataset_Generation_Code_Agentic_pipeline.zip
```
